# 다음 작업 계획

이 문서는 지금까지 구현된 기능을 바탕으로, 앞으로 어떤 순서로 이어가면 좋은지 정리한 다음 단계 계획이다.

## 1. Unity 실연동

- Unity 프로젝트를 준비한다.
- `GET /api/avatar/export/<session_id>` JSON 구조를 읽는다.
- 관절 위치와 회전 값을 Unity 본 구조에 매핑한다.
- 무대/아바타/의상 선택 결과를 실제 렌더링에 연결한다.

## 2. 실제 영상 비교 검증

- 사람 춤 영상 1개를 준비한다.
- MediaPipe 환경과 fallback 환경에서 각각 분석한다.
- `총점`, `동작 유사도`, `불안정 관절`, `구간별 점수`를 비교한다.
- 차이가 잘 드러나는 장면을 문서와 리포트에 기록한다.

## 3. 운영 배포 확인

- 백엔드와 프런트를 운영 설정으로 띄운다.
- 회원가입, 로그인, 연습, 업로드, 관리자 진입을 순서대로 확인한다.
- SSE 스트림이 프록시 환경에서도 끊기지 않는지 본다.

## 4. 추가로 다듬으면 좋은 부분

- 관리자 대시보드에 필터 프리셋 저장을 더 확장한다.
- 리포트 화면의 구간 애니메이션을 조금 더 풍부하게 만든다.
- 업로드 결과를 곡별 프로필과 더 정교하게 연결한다.

## 5. 한 줄 정리

- 지금 단계는 "기능 구현"은 끝났고, 앞으로는 "실연동과 실제 데이터 검증"이 핵심이다.
