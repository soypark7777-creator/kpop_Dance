# MediaPipe vs Fallback 비교

이 문서는 "업로드한 영상이나 프레임을 읽을 때, 실제 MediaPipe가 있을 때와 없을 때 무엇이 다른지"를 초보자도 이해하기 쉽게 설명합니다.

## 1. 먼저 결론

- `MediaPipe`가 있으면 사람의 관절 위치를 실제로 더 정확하게 읽습니다.
- `fallback`은 MediaPipe가 없을 때도 화면이 멈추지 않도록 만든 대체 로직입니다.
- 둘 다 결과를 만들 수 있지만, 정확도와 자연스러움은 MediaPipe 쪽이 더 좋습니다.

## 2. 두 방식의 차이

### MediaPipe 방식

- 영상 프레임을 읽습니다.
- 사람의 관절 위치를 직접 찾습니다.
- 어깨, 팔꿈치, 손목, 골반, 무릎, 발목 같은 좌표를 계산합니다.
- 이 좌표를 기준 안무와 비교해서 점수와 피드백을 만듭니다.

### Fallback 방식

- MediaPipe가 없거나 작동하지 않을 때 사용합니다.
- 이미지 밝기, 프레임 변화, 간단한 패턴을 이용해 "대략적인 자세"를 만듭니다.
- 화면이 아예 비지 않도록 결과를 이어주는 역할입니다.
- 실제 사람 관절을 정확히 찾는 것은 아니기 때문에 정밀도는 낮습니다.

## 3. 왜 fallback이 필요한가

- 개발 환경마다 MediaPipe가 설치되어 있지 않을 수 있습니다.
- 어떤 컴퓨터는 OpenCV만 있고 MediaPipe는 없을 수 있습니다.
- 이럴 때 앱이 멈추지 않게 하려면 대체 경로가 필요합니다.
- 그래서 fallback은 "임시로라도 분석 결과를 보여주는 안전장치"입니다.

## 4. 현재 코드의 동작 흐름

1. 업로드된 영상에서 프레임 이미지를 저장합니다.
2. `pose_service`가 먼저 MediaPipe 사용 가능 여부를 확인합니다.
3. MediaPipe가 있으면 실제 관절 좌표를 읽습니다.
4. MediaPipe가 없으면 fallback 자세를 생성합니다.
5. `angle_service`가 관절 각도를 계산합니다.
6. `dtw_service`가 동작 흐름의 비슷함을 계산합니다.
7. `analysis_service`가 총점, 구간 점수, 피드백을 만듭니다.
8. `report` 화면이 그 결과를 보여줍니다.

## 5. 점수 차이가 나는 이유

- MediaPipe는 실제 관절 좌표를 읽기 때문에 작은 자세 차이도 반영합니다.
- fallback은 대략적인 자세를 사용하므로 세밀한 차이가 덜 보입니다.
- 그래서 같은 영상이라도 MediaPipe 결과가 더 믿을 만합니다.

## 6. 언제 어떤 결과를 믿어야 하나

- 개발 중 화면 흐름 확인: fallback 결과도 충분합니다.
- 실제 연습 점수와 교정: MediaPipe 결과를 기준으로 보는 것이 좋습니다.
- 운영 서비스: MediaPipe가 설치된 환경을 기본으로 잡는 것이 좋습니다.

## 7. 지금 상태

- 코드에는 MediaPipe 우선 로직이 들어가 있습니다.
- MediaPipe가 없으면 fallback으로 자동 전환됩니다.
- 현재 환경에는 MediaPipe도 설치되어 있어서, 실제 포즈 추출 경로를 시험할 수 있습니다.

## 8. 다음 확인 포인트

- 샘플 영상 1개를 MediaPipe 환경과 fallback 환경에서 각각 분석해 비교한다.
- `총점`, `동작 유사도`, `불안정 관절 수`를 나란히 비교한다.
- 차이가 많이 나는 구간이 있는지 확인한다.

## 9. 실제 실행 결과

- 테스트 이미지: `tmp_pose_test.png`
- 실행 환경: MediaPipe 설치됨, OpenCV 설치됨
- MediaPipe 결과
  - 감지된 관절 수: `13`
  - 동작 유사도: `64.19`
  - 불안정 관절 수: `0`
- fallback 결과
  - 감지된 관절 수: `13`
  - 동작 유사도: `64.19`
  - 불안정 관절 수: `0`
- 해석
  - 이번 샘플은 MediaPipe가 별도 포즈를 확정하지 못해 fallback과 같은 결과가 나왔다.
  - 즉, 현재 앱은 MediaPipe가 잘 잡히는 영상에서는 더 정확한 값을 줄 수 있고, 그렇지 않은 영상에서는 fallback으로 안전하게 이어진다.
  - 다음에는 실제 사람 영상 프레임으로 다시 비교하면 차이가 더 잘 보인다.
