# 로그인부터 관리자 진입까지 테스트 플로우

이 문서는 회원가입, 로그인, 관리자 진입이 실제로 이어지는지 초보자 기준으로 확인하는 절차를 정리한 것이다.

## 1. 준비

- 백엔드 실행
- 프런트 실행
- 프런트에서 `NEXT_PUBLIC_MOCK_MODE=false` 로 실제 API 모드 사용
- `NEXT_PUBLIC_API_URL=http://localhost:5000` 확인

## 2. 회원가입

1. 브라우저에서 `/auth`를 연다.
2. `회원가입` 탭을 선택한다.
3. 이메일, 닉네임, 비밀번호, 비밀번호 확인을 입력한다.
4. `회원가입` 버튼을 누른다.
5. 성공하면 토큰이 저장되고 홈으로 이동한다.

## 3. 일반 로그인

1. `/auth`에서 `로그인` 탭을 선택한다.
2. 방금 만든 계정의 이메일과 비밀번호를 입력한다.
3. `로그인` 버튼을 누른다.
4. 성공하면 토큰이 저장되고 홈으로 이동한다.
5. 홈에서 `최근 연습`, `아바타`, `리포트` 버튼이 정상 동작하는지 확인한다.

## 4. 관리자 로그인

기본 관리자 계정은 다음 값을 사용한다.

- 이메일: `admin@kpopdance.local`
- 비밀번호: `change-me-admin`

절차:

1. `/auth`에서 `로그인` 탭으로 이동한다.
2. 위 관리자 계정으로 로그인한다.
3. 성공하면 `/admin`으로 이동한다.
4. 관리자 대시보드가 보이면 성공이다.

## 5. 확인할 것

- `POST /api/auth/register`가 `201`로 응답하는지 확인
- `POST /api/auth/login`이 `success: true`를 반환하는지 확인
- 브라우저 Local Storage에 `kpop_dance_access_token`이 저장되는지 확인
- `/api/auth/me` 호출이 토큰 기반 사용자 정보를 반환하는지 확인
- `/admin`에서 사용자/세션/업로드 목록이 보이는지 확인

## 6. 실패하면 먼저 볼 것

- 비밀번호가 8자 이상인지
- 이메일이 중복되지 않는지
- `NEXT_PUBLIC_MOCK_MODE` 값이 의도한 대로 설정됐는지
- `ADMIN_BOOTSTRAP_EMAIL`, `ADMIN_BOOTSTRAP_PASSWORD` 값이 맞는지
- 브라우저 개발자 도구의 Network 탭에서 응답 코드를 확인했는지
