# Backend Scoring Flow

이 문서는 `kpop_dance` 백엔드가 점수를 계산하고, 분석 리포트를 만들고, 디버그 정보를 보여주는 순서를 초보자도 따라갈 수 있게 정리한 안내서입니다.

## 전체 흐름

```text
사용자 시작
   |
   v
POST /api/session/start
   |
   v
세션 생성
   |
   v
GET /api/stream/live?session_id=...
   |
   v
프레임 수신
   |
   v
1) 포즈 추출
2) 관절 각도 계산
3) 기준 안무와 비교
4) DTW 유사도 계산
5) 피드백 생성
   |
   v
실시간 점수 표시
   |
   v
POST /api/session/end
   |
   v
분석 리포트 생성
   |
   v
GET /api/analysis/<session_id>
   |
   v
리포트 화면 / 관리자 화면
```

## 서비스 역할

### 1. `pose_service.py`

- 입력 프레임에서 사람의 포즈 데이터를 정리합니다.
- 실제 프로젝트에서는 MediaPipe 결과를 받는 자리입니다.
- 현재 구조에서는 JSON 형태의 포즈 데이터를 안전하게 다룹니다.

### 2. `angle_service.py`

- 어깨, 팔꿈치, 손목, 엉덩이, 무릎, 발목의 각도를 계산합니다.
- 사용자와 기준 안무의 각도 차이를 구합니다.
- 차이가 크면 `wrong joint`로 판단할 수 있습니다.

### 3. `dtw_service.py`

- 동작의 흐름이 전체적으로 비슷한지 봅니다.
- 각 프레임을 하나씩 비교하는 것보다, 움직임의 패턴을 더 넓게 평가합니다.
- 결과는 `motion_similarity` 같은 값으로 쓰입니다.

### 4. `feedback_service.py`

- 계산된 결과를 바탕으로 짧은 코치 문구를 만듭니다.
- 예:
  - 팔이 너무 올라갔어요
  - 무릎 각도를 조금 더 맞춰보세요
  - 중심이 흔들렸어요

### 5. `session_service.py`

- 실시간 세션을 만들고 관리합니다.
- 현재 프레임의 점수를 계산합니다.
- 스트림으로 보낼 JSON을 만들어 줍니다.

### 6. `analysis_service.py`

- 세션 전체 결과를 종합합니다.
- 프레임별 점수, 각도 오차, DTW 유사도를 합쳐 리포트를 만듭니다.
- 관리자 화면과 리포트 화면이 보는 핵심 데이터도 여기서 정리됩니다.

### 7. `replay_service.py`

- 세션 동안 쌓인 프레임을 다시 재생할 수 있게 정리합니다.
- 나중에 복습 화면이나 디버그 화면에 연결하기 좋습니다.

### 8. `avatar_export_service.py`

- 분석 결과를 Unity나 외부 렌더러가 읽기 쉬운 JSON으로 바꿉니다.

## 점수 계산 방식

점수는 한 번에 하나만 보지 않고 여러 신호를 합쳐서 만듭니다.

### 기본 생각

```text
기준 점수
  - 관절 각도 차이 벌점
  - 흔들림이 큰 구간 벌점
  - DTW 유사도 보정
  = 최종 점수
```

### 실제 의미

- 관절 각도 차이가 작을수록 점수가 올라갑니다.
- 틀린 관절이 많을수록 점수가 내려갑니다.
- 동작의 전체 흐름이 비슷할수록 점수가 올라갑니다.
- 최종적으로 0~100 사이에서 보기 좋은 값으로 정리합니다.

### 화면에서 보는 대표 값

- `total_score`
- `motion_similarity`
- `average_angle_error`
- `most_wrong_joints`
- `unstable_joint_count`

## 실시간 디버그 모드

스트림은 아래처럼 디버그 값을 추가로 보여줄 수 있습니다.

```text
GET /api/stream/live?session_id=123&debug=true
```

`debug=true`일 때 확인할 수 있는 값:

- 사용자 관절 각도
- 기준 관절 각도
- 오차가 큰 관절 순위

이 기능은 개발자나 관리자가 “왜 점수가 이렇게 나왔는지” 이해할 때 도움이 됩니다.

## 관리 화면에서 보는 필터

관리자 화면은 세션을 점수 기준으로 쉽게 찾을 수 있어야 합니다.

그래서 다음 필터를 넣었습니다.

- 최소 평균 점수
- 최대 평균 오차
- 불안정 관절 이름

이 필터는 세션을 빠르게 찾고, 문제 구간이 많은 학습자를 찾는 데 유용합니다.

## 다음 작업 계획

1. `analysis_service`의 구간별 점수를 더 세분화하기
2. 리포트 화면에 `motion_similarity`와 불안정 관절을 더 눈에 띄게 보여주기
3. 관리자 화면에 세션 요약 카드와 검색 필터를 더 강화하기
4. 실제 MediaPipe 입력과 현재 `pose_service`를 더 촘촘히 연결하기
5. 점수 계산 가중치는 `SCORING_*` 환경변수로 조절하기
6. 곡별, 난이도별로 다른 가중치를 쓸 수 있게 프로필 구조로 확장하기

## 한 줄 정리

이 백엔드는 “포즈를 읽고 -> 각도를 계산하고 -> 기준과 비교하고 -> DTW로 흐름을 보고 -> 점수와 피드백을 만든다”는 순서로 움직입니다.
