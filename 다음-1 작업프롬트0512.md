너는 Next.js 14 + TypeScript 프런트엔드와 Flask + MySQL 백엔드를 다루는 시니어 AI 포즈 분석 풀스택 개발자다.

프로젝트:
K-pop Dance Clone / My Avatar Dance Master

현재 상태:
- NEXT_PUBLIC_MOCK_MODE=false 기준으로 practice → SSE → session end → report → replay 흐름 통합 테스트 완료
- API 필드 불일치, CORS, session_id 흐름 문제 수정 완료
- 현재 SSE는 mock frame을 보내고 있음
- 이제 SSE mock frame을 실제 MediaPipe Pose 분석 결과 기반 흐름으로 교체하려고 한다

이번 작업 목표:
SSE mock frame을 실제 MediaPipe Pose 기반 데이터 흐름으로 교체한다.
단, MVP 1차에서는 “프런트엔드에서 MediaPipe로 웹캠 포즈 추출 → 백엔드로 pose frame 전송 → 백엔드가 기준 안무와 비교/점수화 → SSE로 분석 결과 반환” 구조로 구현한다.

중요 원칙:
1. 백엔드가 직접 사용자의 웹캠을 열지 않는다.
2. 프런트엔드가 브라우저에서 MediaPipe Pose를 실행한다.
3. 백엔드는 프런트에서 받은 user_pose와 dance_reference의 reference_pose를 비교한다.
4. 기존 SSE 연결 구조는 유지한다.
5. 기존 practice → report → replay → avatar 흐름을 깨지 않는다.
6. mock fallback은 삭제하지 않는다.
7. NEXT_PUBLIC_MOCK_MODE=true이면 기존 mock 흐름이 계속 동작해야 한다.
8. NEXT_PUBLIC_MOCK_MODE=false이면 실제 MediaPipe 흐름을 우선 사용한다.
9. Joint 이름은 절대 변경하지 않는다.
10. 모든 일반 HTTP API는 공통 응답 wrapper를 사용한다.
11. SSE event.data는 LiveFrameData를 직접 보낸다.
12. snake_case 필드명을 유지한다.

이번 작업에서 구현할 전체 흐름:

1. practice 페이지에서 카메라 시작
2. 브라우저에서 MediaPipe Pose 실행
3. 각 프레임마다 13개 joint만 추출
4. user_pose를 백엔드로 전송
5. 백엔드는 현재 session_id와 timestamp 기준으로 reference_pose를 찾음
6. user_pose와 reference_pose를 각도 기반으로 비교
7. match_score, joint_errors, feedback 생성
8. 분석 결과를 SSE queue에 넣음
9. 프런트는 기존 EventSource로 LiveFrameData 수신
10. session_frames에 user_pose, reference_pose, score 저장
11. session end 시 저장된 frames 기반 report 생성

4. practice 페이지 동작
- 세션 시작 후 MediaPipe를 시작한다.
- MediaPipe가 pose를 추출하면 일정 FPS로 /api/session/frame에 보낸다.
- 전송 FPS는 우선 10fps 이하로 제한한다.
- SSE는 계속 /api/stream/live?session_id=...를 듣는다.
- SSE로 받은 match_score와 feedback은 기존 UI에 그대로 반영한다.
- MediaPipe 초기화 실패 시 mock fallback 또는 안내 UI를 보여준다.

백엔드 요구사항:

1. POST /api/session/frame 구현
목적:
프런트에서 추출한 user_pose를 받아 분석하고 저장한다.

Request:
{
  "session_id": "session_uuid",
  "timestamp_seconds": 1.23,
  "user_pose": PoseData
}

처리:
- session_id 유효성 확인
- dance_reference_id 조회
- reference_pose 조회
- user_pose와 reference_pose 비교
- match_score 계산
- joint_errors 생성
- feedback 생성
- session_frames에 저장
- 해당 session_id의 SSE queue에 LiveFrameData push

Response:
{
  "success": true,
  "data": {
    "accepted": true,
    "frame_index": 123
  },
  "message": "프레임을 수신했습니다.",
  "timestamp": "..."
}

2. reference_pose_service 구현
MVP에서는 실제 기준 안무 JSON이 부족할 수 있으므로 2단계 fallback 구조로 만든다.

우선순위:
1. dance_reference.reference_json_path에 JSON이 있으면 timestamp_seconds에 가장 가까운 frame을 반환
2. JSON이 없으면 안전한 demo reference_pose를 반환

reference JSON 구조 예:
{
  "dance_id": "neon_step_beginner",
  "fps": 15,
  "frames": [
    {
      "frame_index": 0,
      "timestamp_seconds": 0.0,
      "joints": {
        "left_shoulder": { "x": 0.4, "y": 0.3, "z": 0.0, "visibility": 1.0 }
      }
    }
  ]
}

3. angle_service 구현
- shoulder, elbow, knee 중심으로 각도 계산
- calculate_angle(a, b, c) 구현
- angle_diff >= 15도이면 wrong
- 낮은 visibility 또는 null joint는 skipped 처리
- 계산 불가능한 관절은 에러가 아니라 ignored로 처리

분석할 주요 각도:
- left_elbow: left_shoulder - left_elbow - left_wrist
- right_elbow: right_shoulder - right_elbow - right_wrist
- left_shoulder: left_elbow - left_shoulder - left_hip
- right_shoulder: right_elbow - right_shoulder - right_hip
- left_knee: left_hip - left_knee - left_ankle
- right_knee: right_hip - right_knee - right_ankle
- left_hip: left_shoulder - left_hip - left_knee
- right_hip: right_shoulder - right_hip - right_knee

4. score 계산
간단한 MVP 점수:
- 계산 가능한 관절 angle_diff 평균을 구한다.
- average_error가 0이면 100점
- average_error가 45도 이상이면 40점 이하로 제한
- score = max(40, 100 - average_error * 1.4)
- 프레임 score는 0~100 범위로 clamp

5. feedback_service 구현
joint_errors 중 가장 큰 angle_diff를 가진 관절을 기준으로 한국어 피드백 생성.

예:
- left_elbow wrong → "왼쪽 팔꿈치 각도를 기준 동작에 더 가깝게 맞춰보세요."
- right_knee wrong → "오른쪽 무릎 굽힘이 기준 동작과 달라요."
- 전체 ok → "좋아요. 지금 동작이 기준 안무와 잘 맞고 있어요."

6. SSE stream_routes 수정
현재 mock frame을 직접 생성하는 방식에서 다음 구조로 변경:
- session_id별 in-memory queue를 둔다.
- /api/session/frame에서 분석 결과를 queue에 push한다.
- /api/stream/live는 해당 queue를 읽어 LiveFrameData를 전송한다.
- queue에 데이터가 없으면 heartbeat 또는 마지막 상태를 보낸다.
- 개발 서버 재시작 시 queue는 초기화되어도 된다.
- 운영 영속성은 session_frames DB에 저장한다.

7. session_frames 저장
저장 필드:
- session_id
- frame_index
- timestamp_seconds
- pose_json
- reference_pose_json 가능하면 포함
- score
- joint_errors_json 가능하면 포함

주의:
현재 DB에 reference_pose_json, joint_errors_json 컬럼이 없으면 당장 마이그레이션 추가하지 말고 pose_json 안에 nested로 저장해도 된다.

pose_json 예:
{
  "user_pose": {},
  "reference_pose": {},
  "joint_errors": {},
  "feedback": {},
  "match_score": 82
}

8. session end 수정
- 저장된 session_frames의 score 평균으로 total_score 계산
- most_wrong_joints 집계
- average_angle_error 집계 가능하면 계산
- analysis_reports 생성
- total_score >= 80이면 unlock_avatar_render true
