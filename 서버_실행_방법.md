# Kpop Dance 프로젝트 서버 실행 방법

## 프로젝트 구조

```text
Kpop_Dance_clone/
├─ frontend/          ← Next.js 프런트엔드
├─ backend/           ← Flask 백엔드
├─ kpopMD/            ← 프로젝트 참고 문서
└─ ...
```

---

## 2. 백엔드 서버 실행

현재 백엔드는 Flask 3.x, SQLAlchemy, Flask-Migrate, MySQL 기준으로 구성되어 있습니다.

### 위치

```powershell
cd e:\Kpop_Dance_clone\backend
```

### 사전 요구사항

- Python 3.11 이상
- MySQL 8.x
- database name: `avatar_dance_db`

### 가상환경 활성화

프로젝트 루트에 `.venv`가 있는 구조입니다. `backend` 폴더 기준으로 아래 명령을 사용합니다.

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
```

현재 폴더 안에 `venv`를 따로 만든 경우에만 아래 명령을 사용합니다.

```powershell
.\venv\Scripts\Activate.ps1
```

### 패키지 설치

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 환경변수 설정

```powershell
cd e:\Kpop_Dance_clone\backend
copy .env.example .env
```

`.env` 주요 값:

```env
FLASK_ENV=development
FLASK_APP=run.py
SECRET_KEY=change-me
DATABASE_URL=mysql+pymysql://root:비밀번호@localhost:3306/avatar_dance_db?charset=utf8mb4
CORS_ORIGINS=http://localhost:3000
API_PUBLIC_BASE_URL=http://127.0.0.1:5000
```

## 3. MySQL 초기화

### DB 생성

```powershell
cd e:\Kpop_Dance_clone\backend
mysql -u root -p < sql/init.sql
```

### 테이블 직접 생성

Flask-Migrate를 쓰지 않고 SQL로 바로 만들 경우:

```powershell
cd e:\Kpop_Dance_clone\backend
mysql -u root -p avatar_dance_db < sql/schema.sql
```

### Seed 데이터 입력

`sql/init.sql`에는 기본 유저와 안무 3개가 포함되어 있습니다.

- Neon Step Beginner
- Retro Groove Easy
- Idol Intro Basic

이미 DB와 테이블을 만든 뒤 Flask CLI로 seed를 넣으려면:

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
$env:FLASK_APP="run.py"
flask seed-demo
```

## 4. Flask-Migrate 사용 방법

처음 1회:

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
$env:FLASK_APP="run.py"

flask db init
flask db migrate -m "initial backend api schema"
flask db upgrade
flask seed-demo
```

이후 모델 변경 시:

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
$env:FLASK_APP="run.py"

flask db migrate -m "schema update"
flask db upgrade
```

## 8. 주요 API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/health` | 서버 상태 확인 |
| GET | `/api/dance-references` | 안무 목록 조회 |
| GET | `/api/dance-references/<id>` | 단일 안무 조회 |
| POST | `/api/session/start` | 연습 세션 시작 |
| GET | `/api/stream/live?session_id=<id>` | SSE 실시간 분석 스트림 |
| POST | `/api/session/end` | 세션 종료 및 리포트 생성 |
| GET | `/api/analysis/<session_id>` | 세션 분석 결과 조회 |
| GET | `/api/session/<session_id>/frames` | 리플레이 프레임 조회 |
| GET | `/api/avatar/export/<session_id>` | Unity용 export JSON |

## 9. 빠른 API 테스트

### PowerShell 테스트

헬스체크:

```powershell
Invoke-WebRequest -Uri http://127.0.0.1:5000/api/health -UseBasicParsing
```

안무 목록:

```powershell
Invoke-WebRequest -Uri http://127.0.0.1:5000/api/dance-references -UseBasicParsing
```

세션 시작:

```powershell
Invoke-WebRequest `
  -Uri http://127.0.0.1:5000/api/session/start `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"user_id":1,"dance_reference_id":1}' `
  -UseBasicParsing
```

세션 종료:

```powershell
Invoke-WebRequest `
  -Uri http://127.0.0.1:5000/api/session/end `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"session_id":"위에서_받은_UUID"}' `
  -UseBasicParsing
```

분석 조회:

```powershell
Invoke-WebRequest -Uri http://127.0.0.1:5000/api/analysis/위에서_받은_UUID -UseBasicParsing
```

### Postman 테스트

#### 1. Health

```text
GET http://127.0.0.1:5000/api/health
```

#### 2. Dance References

```text
GET http://127.0.0.1:5000/api/dance-references
```

#### 3. Session Start

```text
POST http://127.0.0.1:5000/api/session/start
```

Body:

```json
{
  "user_id": 1,
  "dance_reference_id": 1
}
```

응답의 `data.session_id` 값을 복사합니다.

#### 4. Session End

```text
POST http://127.0.0.1:5000/api/session/end
```

Body:

```json
{
  "session_id": "복사한_session_id"
}
```

#### 5. Analysis

```text
GET http://127.0.0.1:5000/api/analysis/복사한_session_id
```

## 10. 문제 해결

### 프런트엔드 포트 충돌

```powershell
cd e:\Kpop_Dance_clone\frontend
npx next dev -p 3001
```

### 백엔드 포트 충돌

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
$env:FLASK_APP="run.py"
flask run --host 127.0.0.1 --port 5001
```

### `node_modules` 오류

PowerShell에서는 아래처럼 정리합니다.

```powershell
cd e:\Kpop_Dance_clone\frontend
Remove-Item -Recurse -Force node_modules
npm install
```

### Python 가상환경 활성화 오류

PowerShell 실행 정책 문제일 수 있습니다.

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### `flask_migrate` 모듈이 없다는 오류

가상환경이 활성화되지 않았거나 패키지가 설치되지 않은 상태입니다.

```powershell
cd e:\Kpop_Dance_clone\backend
..\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### MySQL 연결 오류

`.env`의 `DATABASE_URL`을 확인합니다.

```env
DATABASE_URL=mysql+pymysql://root:비밀번호@localhost:3306/avatar_dance_db?charset=utf8mb4
```

그리고 DB가 생성되어 있는지 확인합니다.

```powershell
mysql -u root -p -e "SHOW DATABASES;"
```
