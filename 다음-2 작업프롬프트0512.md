너는 Python 3.11, Flask, MediaPipe, OpenCV, MySQL, SQLAlchemy를 사용하는 시니어 AI 포즈 데이터 엔지니어다.

프로젝트:
K-pop Dance Clone / My Avatar Dance Master

현재 상태:
- 프런트 MediaPipe Pose → /api/session/frame → 백엔드 점수 계산 → SSE 반환 흐름을 구현 중이다.
- 다음 단계로 기준 안무 영상에서 reference pose JSON을 생성하는 도구가 필요하다.
- 이 JSON은 dance_references.reference_json_path에 등록되어 실시간 분석 기준 데이터로 사용된다.

이번 작업 목표:
관리자가 직접 소유했거나 사용 허가를 받은 기준 안무 MP4 파일을 입력하면,
MediaPipe Pose로 프레임별 13개 joint를 추출하고,
실시간 분석에 사용할 기준 안무 JSON 파일을 생성하는 CLI 도구와 관리자 API를 만든다.

중요 법적/보안 원칙:
1. YouTube 무단 다운로드 기능은 구현하지 않는다.
2. DRM 우회, 로그인 우회, 비공개 영상 접근 기능을 만들지 않는다.
3. 로컬에 존재하는 영상 파일 또는 관리자가 직접 업로드한 허가 영상만 처리한다.
4. 관리자 화면에는 “본인이 소유했거나 사용 허가를 받은 영상만 기준 안무로 등록하세요.” 문구를 유지한다.

반드시 지킬 프로젝트 규칙:
1. Joint 이름은 절대 변경하지 않는다.
2. 모든 JSON은 UTF-8, ensure_ascii=False로 저장한다.
3. 좌표는 normalized x, y, z, visibility 구조로 저장한다.
4. 기본 출력 FPS는 15fps로 제한한다.
5. MediaPipe가 못 잡은 프레임은 skipped_frames에 기록한다.
6. 기준 JSON 구조는 reference_pose_service가 바로 읽을 수 있어야 한다.
7. 기존 mock/fallback 구조를 삭제하지 않는다.
8. 기존 DB 테이블 구조를 최대한 유지한다.
9. 필요하면 dance_references.reference_json_path만 업데이트한다.

1. CLI 도구: extract_reference_pose.py

목적:
MP4 영상을 입력받아 MediaPipe Pose로 기준 안무 JSON을 생성한다.

실행 예:
python backend/tools/extract_reference_pose.py \
  --input backend/storage/reference_videos/converted/neon_step.mp4 \
  --output backend/storage/dance_reference/neon_step_beginner.json \
  --dance-id neon_step_beginner \
  --title "Neon Step Beginner" \
  --fps 15 \
  --start 0 \
  --end 16

필수 옵션:
- --input: 입력 영상 경로
- --output: 출력 JSON 경로
- --dance-id
- --title
- --fps 기본 15
- --start 선택, 기본 0
- --end 선택, 기본 영상 끝까지
- --mirror 선택, true면 좌우 반전 보정
- --min-visibility 기본 0.5

출력 JSON 구조:
{
  "schema_version": "1.0",
  "dance_id": "neon_step_beginner",
  "title": "Neon Step Beginner",
  "source_video": "neon_step.mp4",
  "fps": 15,
  "duration_seconds": 16.0,
  "joint_names": [
    "nose",
    "left_shoulder",
    "right_shoulder",
    "left_elbow",
    "right_elbow",
    "left_wrist",
    "right_wrist",
    "left_hip",
    "right_hip",
    "left_knee",
    "right_knee",
    "left_ankle",
    "right_ankle"
  ],
  "frames": [
    {
      "frame_index": 0,
      "timestamp_seconds": 0.0,
      "joints": {
        "left_shoulder": {
          "x": 0.38,
          "y": 0.28,
          "z": -0.02,
          "visibility": 0.99
        }
      }
    }
  ],
  "sections": [
    {
      "section_index": 0,
      "section_name": "intro",
      "start_time": 0.0,
      "end_time": 4.0
    }
  ],
  "stats": {
    "source_total_frames": 480,
    "exported_frames": 240,
    "skipped_frames": 3,
    "average_visibility": 0.94
  },
  "created_at": "ISO datetime"
}

구현 요구:
- OpenCV로 영상 로드
- MediaPipe PoseLandmarker 또는 mediapipe.solutions.pose 사용
- 입력 FPS를 읽고 --fps로 다운샘플링
- start/end 구간만 처리
- 13개 joint만 저장
- visibility 낮은 joint는 null로 저장하거나 visibility 포함
- 프레임 전체 pose가 너무 불완전하면 skipped 처리
- 출력 폴더가 없으면 생성
- 처리 진행률 로그 출력
- 마지막에 통계 출력
- 실패 시 친절한 에러 메시지 출력

2. 검증 도구: validate_reference_json.py

목적:
생성된 기준 안무 JSON이 앱에서 사용할 수 있는지 검증한다.

실행 예:
python backend/tools/validate_reference_json.py \
  --input backend/storage/dance_reference/neon_step_beginner.json

검증 항목:
- schema_version 존재
- dance_id 존재
- fps 존재
- frames 배열이 비어 있지 않음
- 모든 frame에 frame_index, timestamp_seconds, joints 존재
- joint_names가 프로젝트 표준 13개와 일치
- timestamp_seconds가 오름차순
- 좌표 x/y가 0~1 범위 안에 있는지 확인
- visibility가 0~1 범위 안에 있는지 확인
- null joint 비율이 너무 높으면 warning
- 결과를 PASS/WARNING/FAIL로 출력

3. 관리자 API

생성할 API:

A. POST /api/admin/dance-references/<id>/generate-reference-json

목적:
이미 업로드/변환된 reference video를 기준으로 reference JSON을 생성한다.

Request:
{
  "converted_mp4_path": "storage/reference_videos/converted/1_reference.mp4",
  "dance_id": "neon_step_beginner",
  "title": "Neon Step Beginner",
  "fps": 15,
  "start": 0,
  "end": 16,
  "mirror": false
}

처리:
- 관리자 권한 필요
- 파일 존재 확인
- extract service 실행
- output path 생성:
  storage/dance_reference/{dance_id}.json
- validate_reference_json 실행
- dance_references.reference_json_path 업데이트
- 결과 반환

Response:
{
  "success": true,
  "data": {
    "dance_reference_id": 1,
    "reference_json_path": "storage/dance_reference/neon_step_beginner.json",
    "stats": {
      "exported_frames": 240,
      "skipped_frames": 3,
      "average_visibility": 0.94
    },
    "validation": {
      "status": "PASS",
      "warnings": []
    }
  },
  "message": "기준 안무 JSON이 생성되었습니다.",
  "timestamp": "..."
}

B. GET /api/admin/dance-references/<id>/reference-json/validate

목적:
등록된 reference_json_path의 JSON을 다시 검증한다.

Response:
{
  "success": true,
  "data": {
    "status": "PASS",
    "warnings": [],
    "errors": []
  },
  "message": "기준 JSON 검증 완료",
  "timestamp": "..."
}

4. reference_pose_service 연결

현재 실시간 분석에서 reference_pose_service가 demo reference_pose를 fallback으로 쓰고 있다면,
다음 순서로 고친다.

우선순위:
1. dance_references.reference_json_path가 존재하고 JSON 파일이 유효하면 timestamp_seconds에 가장 가까운 frame 반환
2. 없거나 실패하면 demo reference_pose 반환
3. 실패해도 session/frame API가 crash하지 않게 처리

필수 함수:
- load_reference_json(dance_reference_id)
- get_reference_pose_at(dance_reference_id, timestamp_seconds)
- find_nearest_frame(frames, timestamp_seconds)
